# fix-e113a8425c120e51 — Redis key prefix mismatch between E2T1 (/fav) and E3T1 (/addfav, /favs, /delfav) — data siloed

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 CVB

`src/handlers/E2T1.ts:6` uses `FAV_KEY_PREFIX = "convertbuddy:favs:"` (matching the spec `docs/spec.md`). `src/handlers/E3T1.ts:6` and `src/handlers/favs.ts:5` use `FAV_KEY_PREFIX = "favs:"`. Result: favorites saved via `/addfav` go to `favs:{userId}`, while favorites saved via `/fav` go to `convertbuddy:favs:{userId}`. `/favs` (served by E3T1) reads from `favs:`, so a user who adds a favorite via `/fav` will **never** see it listed in `/favs`. The two feature handlers operate on entirely separate Redis keys.
